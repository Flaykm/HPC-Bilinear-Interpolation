## Задача

Реализовать алгоритм билинейной интерполяции для увеличения размера изображения в два раза:
- **Входные данные**: Исходное изображение размера $M \times N$.
- **Реализация**: 
    - Сравнение производительности CPU и GPU (CUDA).
    - Обработка краевых эффектов: недостающие пиксели для последних строк и столбцов берутся с противоположного края изображения.
    - Использование CUDA-ядер для параллельного вычисления значений новых пикселей.
- **Выходные данные**: Итоговое изображение $2M \times 2N$, время выполнения на CPU и GPU, график ускорения.

---

## Реализация

В работе реализован алгоритм интерполяции, использующий весовые коэффициенты соседних пикселей для формирования нового кадра. Основные вычислительные этапы перенесены на GPU с использованием библиотеки `numba.cuda`.

- **CPU** (`bilinear_interp_cpu`) — векторизованная реализация с использованием `numpy.meshgrid` и `numpy.floor`. Для обработки границ применяется оператор взятия остатка `%`, что соответствует режиму *wrapping*.
- **GPU Kernel** (`bilinear_interp_kernel`) — параллельная реализация, где каждый поток GPU вычисляет один пиксель результирующего изображения.
    - **Интерполяция**: Вычисляются координаты в исходном пространстве, определяются четыре ближайших соседа и их веса ($tx, ty$).
    - **Boundary Handling**: Реализован механизм *wrap* (циклическое повторение), позволяющий корректно вычислять значения даже для пикселей, выходящих за границы исходной сетки.
    - **Memory**: Использование `cuda.to_device` и `device_array` для минимизации задержек при работе с памятью.

## Выводы

1. GPU показывает колоссальное преимущество на задачах обработки изображений. На максимальном протестированном разрешении (1024x1024) обработка на GPU прошла почти в 150 раз быстрее, чем на CPU.
2.  С ростом объема данных (количества пикселей) эффективность распараллеливания через CUDA возрастает, так как накладные расходы на запуск ядра становятся незначительными относительно времени вычислений.
